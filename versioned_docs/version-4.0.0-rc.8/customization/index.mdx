Congrats! You've installed `@umar-epacken/salon-data` and your immediate question
goes something like this:

> So umm, how do can i get the user information through the redux

Great question! you can get the current user information through `useHUser()` hook

## useHUser()

you can get the current user context by utilizing `useHUser`

```jsx
import { useHUser } from "@umar-epacken/salon-data";

const { loginInfo } = useHUser();

console.log(loginInfo); /** current user info **/
```

# Application Hooks

## 1. useAppConfig

**Platform:** WEB/MOBILE

**Usage:** Access application configuration.

**Returns:**

- `loading`: Application loading state.
- `language`: Application language.
- `themeMode`: Application theme mode (`light` | `dark`).
- `accessToken`: Application access token.
- `userInfo`: Current logged-in user info.
- `setReduxLanguage`: Public setter method for language.
- `setReduxThemeMode`: Public setter method for application theme mode.

## 2. useRegister

**Platform:** WEB/MOBILE

**Usage:** Sign up/register user to the application.

**Parameters:**

- `username`: string
- `email`: string
- `password`: string

**Returns:**

- `userInfo`: Current registered user info.

## 3. useLogin

**Platform:** WEB/MOBILE

**Usage:** Sign in/login user to the application.

**Parameters:**

- `identifier`: string
- `password`: string
- `provider`: string

**Returns:**

- `userInfo`: Current logged-in user info.

## 4. useOtp

**Platform:** WEB/MOBILE

**Usage:** Phone number verification via OTP.

**Parameters:**

- `otpCode`: string
- `otpReceiveType`: _ENUM_ EOtpReceive (`SMS` | `EMAIL`)

**Returns:**

- `OtpAPI`: Public method to get OTP.
- `validateOtpAPI`: Public method to validate OTP.

## 5. useSession

**Platform:** WEB/MOBILE

**Usage:** Validate the current application session by verifying JWT.

**Parameters:**

- `accessToken`: string

**Returns:**

- `validateSessionAPI()`: Public method to validate token and update in redux.

## 6. useForgotPassword

**Platform:** WEB/MOBILE

**Usage:** Send reset password email to user.

**Parameters:**

- `email`: string

**Returns:**

- `forgotPasswordAPI()`: Public method to access API and get response.

## 7. useResetPassword

**Platform:** WEB/MOBILE

**Usage:** Reset user password.

**Parameters:**

- `password`: string
- `passwordConfirmation`: string
- `code`: string

**Returns:**

- `resetPasswordAPI()`: Public method to access resetPassword API and get response.

## 8. useChangePassword

**Platform:** WEB/MOBILE

**Usage:** Change current user password.

**Parameters:**

- `currentPassword`: string
- `password`: string
- `passwordConfirmation`: string

**Returns:**

- `changePasswordAPI()`: Public method to access changePassword API and get response.

## 9. useEmailConfirmation

**Platform:** WEB/MOBILE

**Usage:** Confirm user email.

**Parameters:**

- `confirmation`: string

**Returns:**

- `emailConfirmationAPI()`: Public method to access emailConfirmation API and get response.

Remember if you want to override the values set in the theme you can always use component props.
